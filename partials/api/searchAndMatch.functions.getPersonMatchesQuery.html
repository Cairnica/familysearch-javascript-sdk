getPersonMatchesQuery (functions in module searchAndMatch )

Description

Get matches for someone not in the tree The response includes the following convenience function

  • getSearchResults() - get the array of SearchResults from the response
  • getResultsCount() - get the total number of search results
  • getIndex() - get the starting index of the results array

Usage

getPersonMatchesQuery(params[, opts]);

Parameters

  • params – {Object} –

    same parameters as described for getPersonSearch, with the exception that context is not a valid parameter for match, and candidateId restricts matches to the person with that Id

  • opts(optional) – {Object=} –

    options to pass to the http function specified during init

Returns

{Object} –

promise for the response
