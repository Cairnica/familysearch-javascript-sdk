getMultiDiscussion (functions in module discussions )

Description

Get multiple discussions at once by requesting them in parallel

FamilySearch API Docs

editable example

Usage

getMultiDiscussion(dids[, params][, opts]);

Parameters

  • dids – {Array} –

    Ids of the discussions to read

  • params(optional) – {Object=} –

    pass to getDiscussion currently unused

  • opts(optional) – {Object=} –

    pass to the http function specified during init

Returns

{Object} –

promise that is fulfilled when all of the discussions have been read, returning a map of discussion id to response
