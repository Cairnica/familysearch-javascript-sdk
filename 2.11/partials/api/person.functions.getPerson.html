getPerson
functions in module person

Description

Get the specified person The response includes the following convenience functions:

  • getPerson() - get the Person from the response
  • getPrimaryId() - id of the person returned
  • getRequestedId() - person id that was requested; may differ from primary id when the requested id was deleted due to a merge
  • getFatherIds() - array of ids
  • getMotherIds() - array of ids
  • getSpouseIds() - array of ids
  • getChildIds() - array of ids of all children
  • getChildIdsOf(spouseId) - array of ids; if spouseId is null/undefined, return ids of children without the other parent
  • getParentRelationships() - array of ChildAndParents relationship objects
  • getSpouseRelationships() - array of Couple relationship objects
  • getSpouseRelationship(spouseId) - Couple relationship with the specified spouse
  • getChildRelationships() - array of ChildAndParents relationship objects
  • getChildRelationshipsOf(spouseId) - array of ChildAndParents relationship objects if spouseId is null/undefined, return ids of child relationships without the other parent
  • getPrimaryPerson() - Person object for the primary person
  • wasRedirected() - returns true when the primary id is different from the requested id

When the relatives parameter is set, the following functions are also available:

  • getPerson(id) - Person object for the person with id
  • getFathers() - array of father Persons
  • getMothers() - array of mother Persons
  • getSpouses() - array of spouse Persons
  • getChildren() - array of all child Persons;
  • getChildrenOf(spouseId) - array of child Persons; if spouseId is null/undefined, return children without the other parent

When the sourceDescriptions parameter is set, the following functions are also available:

  • getSourceDescriptions() get an array of SourceDescriptions from the response
  • getSourceDescription(id) get the SourceDescription with the specified source description id from the response
